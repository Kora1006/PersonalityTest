# Taro 微信小程序迁移 — 需求规格

## 概述

将现有 React Web 应用通过 Taro 4 框架迁移至微信小程序平台，确保核心功能完整运行，同时满足微信小程序的包体积和平台限制。

## 项目信息

- 项目名: PersonalityTest
- 架构类型: Monorepo（npm workspaces + Turborepo）

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-15 | v1   | 初始需求 |

## 用户故事

- 作为用户，我希望在微信内直接打开小程序完成 DISC 测评，以便无需下载 App 或跳转浏览器
- 作为运营，我希望产品上架微信小程序平台，以便借助微信社交生态获取用户增长
- 作为开发者，我希望最大化复用现有 React + TypeScript 代码，以便降低迁移维护成本

## 功能需求

1. [F-001] 使用 Taro 4（React 语法）搭建微信小程序项目结构，复用现有 TypeScript 业务逻辑
2. [F-002] 迁移首页（Home）：Hero 区、DISC 维度介绍 Bento 卡片、Stats 区
3. [F-003] 迁移答题页（Quiz）：24 题 / 进度条 / 单选卡片 / 底部操作条
4. [F-004] 迁移结果页（Result）：雷达图（Canvas 实现）/ 百分比条 / 性格画像
5. [F-005] 迁移深度解析页（Detail）：优势卡片 / 沟通风格 / 成长矩阵
6. [F-006] 迁移历史记录页（History）：本地存储 + 搜索过滤 + 删除
7. [F-007] 接入微信登录（openid 获取 + unionid 绑定）
8. [F-008] 将 SVG 雷达图重构为 Canvas API 实现（小程序不支持原生 SVG 动画）
9. [F-009] 配置分包策略，控制主包 < 2MB，分包总量 < 20MB
10. [F-010] 微信小程序审核合规处理（心理测评类内容措辞、隐私政策页、用户协议页）

## 非功能需求

- 性能: 首屏 TTI（可交互时间）< 2s（4G 网络）
- 包体积: 主包 < 2MB，分包总计 < 20MB
- 兼容性: 微信基础库 >= 3.0.0，iOS 14+ / Android 10+
- 安全: 微信 openid/unionid 仅在服务端存储，前端不持久化

## 验收标准

- [ ] [AC-001] 小程序在微信开发者工具中无报错启动，首页正常渲染
- [ ] [AC-002] 完整走通 24 题答题流程，结果页雷达图（Canvas）正常显示动画
- [ ] [AC-003] 历史记录可正常保存、查询、删除（使用微信 `wx.setStorageSync`）
- [ ] [AC-004] 微信登录流程：授权 → 获取 openid → 写入服务端用户表，全链路无报错
- [ ] [AC-005] 主包体积经 `npm run build:weapp` 后 < 2MB
- [ ] [AC-006] 审核合规：隐私政策页、用户协议页存在且可访问；测评类文案不含医疗诊断措辞
- [ ] [AC-007] 真机测试（iOS + Android）核心流程无 crash，雷达图渲染正常

## 功能范围

**In Scope:**
- Taro 4 项目初始化与 Monorepo 集成
- 6 个核心页面迁移（首页、答题、结果、深度解析、历史记录、登录）
- Canvas 雷达图重构（替代 SVG）
- 微信登录接入
- 分包配置
- 合规页面（隐私政策、用户协议）

**Out of Scope:**
- 支付宝/百度/字节小程序适配（仅微信）
- H5 Web 版本维护（迁移后并行维护策略另行决策）
- 小程序内置分享（在 7.viral-growth-loop 中实现）
- 多主题皮肤（在 8.disc-multi-theme 中实现）

## 依赖

- Taro 4.x (`@tarojs/taro`, `@tarojs/react`)
- 微信开发者工具 >= 1.06
- 微信小程序 AppID（需向微信开放平台申请）
- 服务端 Better Auth 微信登录插件适配
- `packages/db` schema 新增 `wechat_openid` / `wechat_unionid` 字段

## 风险与假设

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| SVG → Canvas 雷达图重构工作量超预期 | 延迟上线 | 提前 PoC 验证 Canvas 动画可行性 |
| 微信审核「心理测评」类内容被拒 | 上线阻断 | 提前研究竞品过审文案，避免使用「诊断」「评估」等敏感词汇 |
| Taro 4 与现有 shadcn/ui 组件不兼容 | 大量重写 | shadcn/ui 仅用于 Web，小程序端使用原生 Taro 组件 + TailwindCSS for Taro |
| 分包体积超标 | 功能裁剪 | 提前分析依赖体积，按页面分包 |
| 微信登录需企业资质 | 开发者账号无法测试 | 使用测试号 + Mock openid 先行开发 |
