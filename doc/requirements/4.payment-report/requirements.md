# payment-report — 需求规格

## 概述

实现付费解锁深度报告流程：支付收银台（微信支付 + 支付宝）→ 支付状态同步 → 解锁后下载 PDF 报告。

## 项目信息

- 项目名: PersonalityTest
- 架构类型: Turborepo Monorepo（npm workspaces）

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-14 | v1   | 初始需求 |

## 用户故事

- 作为用户，我想要通过微信支付或支付宝购买深度报告，以便获得完整的性格分析材料。
- 作为已付费用户，我想要随时下载 PDF 报告，以便留存或分享给他人。
- 作为运营方，我想要在收到支付回调后自动解锁用户报告，以便实现无人工介入的自动化履约。

## 功能需求

### 支付前置判断
1. [F-001] 用户在深度解析页点击"下载完整报告"时，判断是否已登录，未登录则跳转至登录页（携带 redirect 参数）
2. [F-002] 登录后判断该次测评记录是否已付费（`is_paid` 字段），已付费直接进入下载流程

### 支付收银台
3. [F-003] 弹出支付 Modal，展示：
   - 商品名：DISC 职业性格 30 页深度分析报告（电子版）
   - 原价与限时特惠价
   - 5 分钟支付倒计时
   - 支付方式切换 Tab（微信支付 / 支付宝）
4. [F-004] 前端请求后端接口生成动态支付二维码并展示
5. [F-005] 展示状态文案："请使用微信/支付宝扫码支付，支付成功后自动解锁下载"

### 支付状态同步
6. [F-006] 前端每 3 秒轮询后端接口查询订单支付状态
7. [F-007] 支付成功后，Modal 平滑淡出，界面替换为"下载 PDF 报告"按钮

### 支付回调（Webhook）
8. [F-008] 后端实现微信支付回调端点，验签后将订单 `is_paid` 置为 `true`
9. [F-009] 后端实现支付宝回调端点，验签后同步更新订单状态

### PDF 报告生成与下载
10. [F-010] 后端根据用户测评数据生成 PDF 报告，包含：行为盲点、管理建议、团队协作动态
11. [F-011] PDF 报告提供安全的下载链接（需验证用户 Token + 支付状态）

## 非功能需求

- **安全**: 支付签名必须在后端完成，前端不得明文存储任何密钥或订单敏感信息
- **幂等性**: 同一订单多次回调仅处理一次（幂等去重）
- **超时处理**: 倒计时 5 分钟结束后，收银台自动关闭
- **PDF 生成**: 服务端渲染，不依赖客户端环境
- **下载安全**: PDF 下载链接需校验用户身份和 `is_paid` 状态

## 验收标准

- [ ] [AC-001] 未登录用户点击"下载报告"跳转至登录页，登录后返回深度解析页
- [ ] [AC-002] 未付费时弹出支付 Modal，展示商品信息和二维码
- [ ] [AC-003] 支付成功后 3 秒内 Modal 自动关闭并展示下载按钮
- [ ] [AC-004] 已付费用户再次访问深度解析页，直接展示"下载 PDF 报告"按钮
- [ ] [AC-005] PDF 下载链接在未登录或未付费时返回 403
- [ ] [AC-006] 微信支付和支付宝回调均通过签名验证
- [ ] [AC-007] 同一订单的重复回调不会重复更新数据库

## 功能范围

**In Scope:**
- 支付 Modal（微信支付 + 支付宝二维码）
- 支付状态轮询
- 微信/支付宝 Webhook 处理
- PDF 报告服务端生成（基于测评数据）
- 已付费后的安全下载

**Out of Scope:**
- 退款流程
- 订单列表/账单管理页
- 发票开具

## 依赖

- Feature 3 (`user-auth`) — 登录态 + 用户 ID
- Feature 1 & 2 — 测评结果数据（用于生成 PDF 内容）
- 微信支付商户号（`WECHAT_PAY_MCH_ID`、`WECHAT_PAY_MCH_KEY` 等）
- 支付宝开放平台（`ALIPAY_APP_ID`、`ALIPAY_PRIVATE_KEY` 等）
- `@react-pdf/renderer` — 服务端 PDF 生成
- 新增 DB 表：`orders`（id, userId, assessmentId, amount, status, isPaid, createdAt）
