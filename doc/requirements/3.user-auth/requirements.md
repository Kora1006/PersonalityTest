# user-auth — 需求规格

## 概述

实现用户账户体系：邮箱/密码本地登录注册 + 微信扫码（PC）/ H5 自动授权（移动端）登录，并在登录后将本地历史记录同步至云端数据库。

## 项目信息

- 项目名: PersonalityTest
- 架构类型: Turborepo Monorepo（npm workspaces）

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-14 | v1   | 初始需求 |

## 用户故事

- 作为新用户，我想要通过邮箱注册账号，以便保存跨设备的测评历史。
- 作为已有账号的用户，我想要通过微信扫码快速登录，以便免去记忆密码的麻烦。
- 作为移动端用户，我想要在微信内置浏览器中自动完成登录，以便获得无缝体验。
- 作为已登录用户，我想要本地历史记录自动同步至服务器，以便换设备后不丢失数据。

## 功能需求

### 注册
1. [F-001] 注册表单：邮箱、密码、确认密码字段
2. [F-002] 输入校验：邮箱格式校验、密码强度（至少 6 位，含数字和字母）、两次密码一致校验

### 本地登录
3. [F-003] 登录表单：邮箱 + 密码
4. [F-004] 记住我：勾选后维持长期会话（通过 Better Auth 的持久 Cookie/Token）
5. [F-005] 忘记密码：发送邮件重置链接流程

### 微信登录（PC 扫码）
6. [F-006] 生成微信授权二维码（通过微信开放平台 OAuth2）
7. [F-007] 前端轮询或 WebSocket 监听扫码状态，授权成功后写入 Token 并跳转
8. [F-008] 首次微信登录自动创建账号；后续登录直接关联已有账号

### 微信登录（H5 / 移动端）
9. [F-009] 微信内置浏览器中自动触发 OAuth 2.0 授权页或静默授权
10. [F-010] 后端解析 `code` 获取 `openid` / `unionid`，完成登录或绑定

### 登录后数据同步
11. [F-011] 登录成功后，将 localStorage 中的 `disc_history` 异步同步至服务端数据库
12. [F-012] 后续测评完成后，若已登录则同时写入 localStorage 和服务端

### 权限控制
13. [F-013] 未登录用户访问需要认证的页面时（如支付页），强制跳转至登录页
14. [F-014] 登录后展示用户头像/昵称（Header 右上角 UserMenu）

## 非功能需求

- **安全**: JWT 签名，支持过期与 Refresh Token 机制（由 Better Auth 管理）
- **WeChat**: 需要微信开放平台 AppID + AppSecret（通过环境变量注入）
- **密码**: 服务端哈希存储（由 Better Auth 处理）
- **会话**: 支持多设备同时在线

## 验收标准

- [ ] [AC-001] 邮箱注册成功后可直接登录
- [ ] [AC-002] 格式不合法的邮箱或弱密码在提交前给出即时反馈
- [ ] [AC-003] 登录后 Header 展示用户信息，并有登出入口
- [ ] [AC-004] PC 端微信二维码可正常展示，扫码授权后自动跳转
- [ ] [AC-005] 微信 H5 环境下点击登录自动跳转授权页，回调后完成登录
- [ ] [AC-006] 首次微信登录自动创建账号
- [ ] [AC-007] 登录成功后本地 localStorage 历史记录同步至服务端（不重复插入）
- [ ] [AC-008] 忘记密码发送重置邮件后 60 秒内不可重复发送

## 功能范围

**In Scope:**
- 邮箱/密码注册与登录
- 微信 PC 扫码登录
- 微信 H5 自动授权登录
- 忘记密码（邮件重置）
- 登录后历史同步

**Out of Scope:**
- 手机号/短信验证码登录（PRD 提到但优先级低于微信）
- 第三方 OAuth（GitHub、Google 等）
- 微信绑定手机号引导流程

## 依赖

- `packages/auth` — Better Auth v1.6 配置入口
- `packages/db` — 用户表（auth schema 已有）、历史记录云端表（需新增）
- 微信开放平台账号（AppID、AppSecret）
- 邮件服务（SMTP / Resend 等）用于密码重置
- 新增环境变量：`WECHAT_APP_ID`、`WECHAT_APP_SECRET`、`SMTP_*` 或 `RESEND_API_KEY`
