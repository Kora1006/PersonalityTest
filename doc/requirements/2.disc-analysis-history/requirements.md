# disc-analysis-history — 需求规格

## 概述

实现深度解析页（性格完整报告）和历史记录管理页（localStorage 持久化 + 搜索 + 删除）。

## 项目信息

- 项目名: PersonalityTest
- 架构类型: Turborepo Monorepo（npm workspaces）

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-14 | v1   | 初始需求 |

## 用户故事

- 作为用户，我想要查看深度性格报告（优势/职场表现/沟通风格/成长建议），以便获得可执行的自我提升路径。
- 作为用户，我想要浏览和搜索历史测评记录，以便对比不同时期的性格变化。
- 作为用户，我想要删除不需要的历史记录，以便保持列表整洁。

## 功能需求

### 深度解析页 (Detail)
1. [F-001] 主导类型概览：代表性图标（D=闪电/I=社交/S=天平/C=对勾）+ 维度得分强度进度条
2. [F-002] 核心优势卡片：结合图标列出高阶实力（每种类型 3-4 条）
3. [F-003] 职场表现卡片：协作偏好、管理风格、对微观管理的态度
4. [F-004] 沟通风格双向面板：
   - Express（你的表达方式）：如何向下/向同级传递信息
   - Receive（如何与你沟通）：他人与你对话的黄金法则
5. [F-005] 成长机会矩阵：三卡片网格，针对主导性格瓶颈的微习惯建议
6. [F-006] 报告下载 CTA：展示"下载完整报告"按钮，点击触发支付流程（feature 4 实现）；v1 阶段展示为引导占位按钮

### 历史记录页 (History)
7. [F-007] 历史记录列表：展示每条记录的日期、主导类型、维度得分概况、备注
8. [F-008] 搜索栏：实时过滤，支持性格类型字母（D/I/S/C）、中文名（支配型/稳健等）、日期、备注内容
9. [F-009] 清除搜索：搜索框内一键清除按钮（×）
10. [F-010] 空状态提示：无匹配时展示空状态插图 + 提示文案
11. [F-011] 查看详情：点击记录卡片内"查看详情"→ 载入该记录数据并跳转结果页
12. [F-012] 删除记录：点击删除按钮 → 二次确认弹窗 → 确认后从 localStorage 移除并刷新列表
13. [F-013] 默认 Mock 数据：首次无数据时自动写入 3 条经典模拟数据（涵盖 D/I/S 三种类型）

## 非功能需求

- **数据结构**: 每条历史记录包含 `{ id, date, dominantType, scores: {D,I,S,C}, note }`
- **localStorage 键名**: `disc_history`
- **搜索响应**: 实时过滤（无需防抖，数据量小）
- **响应式**: Mobile / Tablet / PC 全适配

## 验收标准

- [ ] [AC-001] 深度解析页展示当前测评结果的全量性格报告（基于 feature 1 的结果数据）
- [ ] [AC-002] 深度解析页的报告内容因主导类型（D/I/S/C）不同而差异化展示
- [ ] [AC-003] 历史记录页首次访问自动插入 3 条 mock 数据
- [ ] [AC-004] 搜索框输入关键词后实时过滤列表
- [ ] [AC-005] 无搜索结果时展示空状态插图
- [ ] [AC-006] 点击历史记录"查看详情"可复现该次测评结果页
- [ ] [AC-007] 删除操作需二次确认弹窗，确认后实时从列表移除

## 功能范围

**In Scope:**
- 深度解析页完整报告（基于本地结果数据）
- localStorage 历史记录管理
- 搜索过滤与删除

**Out of Scope:**
- 报告实际下载/PDF 生成（feature 4）
- 历史记录云端同步（feature 3）
- 登录后的服务端历史接口

## 依赖

- Feature 1 (`disc-quiz-flow`) — 提供结果数据结构与 localStorage 写入
- `packages/ui` — 共享组件（Dialog、Input、Card 等）
