# DISC 多主题皮肤 — 需求规格

## 概述

基于同一套 DISC 题目和算法，通过「主题皮肤」机制产出多个差异化测评入口（情感版、管理版），每个主题拥有独立的文案包装和结果页视觉，无需重写算法或题目。

## 项目信息

- 项目名: PersonalityTest
- 架构类型: Monorepo（npm workspaces + Turborepo）

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-15 | v1   | 初始需求 |

## 用户故事

- 作为普通用户，我希望通过「你在亲密关系中是哪种人」入口参与测评，以便用更贴近自己生活的视角了解性格
- 作为管理者，我希望通过「你是哪种 Leader」入口参与测评，以便发现自己的领导风格
- 作为运营，我希望不同主题的分享卡和文案各具特色，以便在不同场景下触达不同人群
- 作为开发者，我希望新主题只需配置文案，无需改动核心算法，以便快速扩展主题数量

## 功能需求

### A. 主题系统架构

1. [F-001] 测评流程支持 `theme` 参数：`theme=professional`（职场，现有）/ `theme=relationship`（情感）/ `theme=leadership`（管理）
2. [F-002] 不同主题共享同一套题目（24 题），但题目引导语措辞根据主题调整（如「在工作中」改为「在感情里」）
3. [F-003] 结果页文案根据 `theme` 渲染不同版本（类型名称、金句、深度解析内容）
4. [F-004] 分享卡根据主题使用不同配色和标题文案
5. [F-005] 首页新增「选择测评主题」模块，展示各主题入口卡片

### B. 情感版（relationship）

6. [F-006] 入口文案：「你在亲密关系中是哪种人」
7. [F-007] 题目引导语前缀：「在感情生活中，当...」
8. [F-008] 结果类型名称重映射：
   - D → 主导型恋人（主动推进、决断力强）
   - I → 热情型恋人（浪漫、情感表达丰富）
   - S → 陪伴型恋人（稳定、包容、安全感强）
   - C → 细心型恋人（体贴细节、重视承诺）
9. [F-009] 深度解析文案聚焦：恋爱模式 / 与你相处的方式 / 关系中的成长点
10. [F-010] 分享卡配色：玫瑰金主题，金句如「你是感情里最懂得给对方空间的人」

### C. 管理版（leadership）

11. [F-011] 入口文案：「你是哪种 Leader」
12. [F-012] 题目引导语前缀：「作为一名管理者，当...」
13. [F-013] 结果类型名称重映射：
    - D → 强势型 Leader（目标导向、推动变革）
    - I → 感召型 Leader（擅长激励、凝聚团队士气）
    - S → 教练型 Leader（耐心培养、建立信任）
    - C → 系统型 Leader（流程严谨、数据驱动）
14. [F-014] 深度解析文案聚焦：领导风格 / 团队管理策略 / 最适合的组织文化
15. [F-015] 分享卡配色：深蓝商务主题，金句如「你是那个把团队带向北极星的人」

### D. 主题间关联

16. [F-016] 用户完成任意主题测评后，结果页推荐「查看你在其他场景的风格」入口，引导测完其他主题
17. [F-017] 历史记录区分显示主题标签（职场版 / 情感版 / 管理版）

## 非功能需求

- 扩展性: 新增主题只需新增一份文案配置文件，无需改动算法或页面逻辑
- 性能: 主题切换不引发额外网络请求（文案本地配置）
- 一致性: 三个主题的雷达图计算逻辑完全相同

## 验收标准

- [ ] [AC-001] 从「情感版」入口进入，题目引导语正确显示「在感情生活中，当...」前缀
- [ ] [AC-002] 情感版结果页类型名称显示为「热情型恋人」等情感专属名称，非「I型」职场名称
- [ ] [AC-003] 管理版结果页展示领导风格相关文案，与职场版内容明显差异
- [ ] [AC-004] 历史记录中不同主题记录有对应标签区分
- [ ] [AC-005] 新增主题只需添加配置文件，无需修改页面组件代码（验证扩展性）
- [ ] [AC-006] 三个主题的 DISC 得分计算结果完全一致（相同作答 → 相同分数）

## 功能范围

**In Scope:**
- 主题系统架构（theme 参数 + 文案配置）
- 情感版（relationship）文案包
- 管理版（leadership）文案包
- 首页主题选择入口
- 历史记录主题标签
- 跨主题推荐引导

**Out of Scope:**
- 不同主题使用不同题目（题目一套到底）
- 主题特定的额外维度或算法
- 付费主题（所有主题免费，深度报告由邀请解锁）
