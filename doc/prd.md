## 1. 项目概述
### 1.1 项目背景
在职场与团队协作中，了解个体的行为特质与沟通风格对于提升团队效率、优化岗位配置、以及个人职业生涯发展至关重要。DISC 测评是全球广泛采用的行为风格测评工具，通过测试支配性（Dominance）、影响性（Influence）、稳健性（Steadiness）和服从性（Conscientiousness）四个维度，帮助用户全面认知自我的行为特质。

### 1.2 产品定位
本项目是一款专为职场人、HR及管理层打造的高美感、流畅交互的 DISC 职业性格测评应用。产品提供 24 道精选情境测试题，辅以多维度数据可视化图表（如雷达图）以及深度的性格报告，同时支持本地历史记录存储与智能检索。在 v2.0 中，产品引入了用户账户系统与商业变现闭环，通过付费解锁专业深度报告。

---

## 2. 目标用户与使用场景
- **职场个人**：评估自身工作风格，了解核心优势，寻找职业能力提升和沟通优化路径。
- **团队管理者**：在团队构建或重组时，评估团队内部角色分布，用于能力互补及沟通机制的优化。
- **HR 招聘人员**：在候选人筛选与面试过程中，辅助了解其性格倾向与岗位的契合度。

## 3. 功能需求规格

### 3.1 首页模块 (Home Screen)
#### 3.1.1 页面布局与视觉呈现
- **Hero 主视觉区**：展示极具冲击力的文案（“解码行为特质，放大职场影响力”）及行动点（“开始测评”按钮与“查看历史记录”按钮）。
- **四大核心维度 Bento 磁贴区**：以网格卡片形式分别介绍 **D（支配型）**、**I（影响型）**、**S（稳健型）**、**C（服从/谨慎型）**，附带专属渐变图标。
- **价值展示区 (Benefits Section)**：说明了解性格画像在“沟通效果”、“职业生涯”和“团队动力”三方面的核心价值，配有团队协作等插图。
- **用户信任与数据指标区 (Stats Section)**：以数据大字展示产品运营数据指标（如“1.5万+ 已分析画像”、“98% 准确度评分”、“10分钟 平均测试时长”）。
- **全局悬浮按钮 (FAB)**：在桌面和移动端右下角常驻快速开始测评按钮，并带有“立即开始测评”的 Tooltip 浮窗。

#### 3.1.2 交互逻辑
- 点击“开始测评”，重置答题状态并跳转至测评答题页面。
- 点击“查看历史记录”，跳转至历史记录管理页面。

---

### 3.2 测评答题模块 (Quiz Screen)
#### 3.2.1 测评规则与算法
- **题目总量**：共 24 道题。
- **题目维度分类**：包含职场动力 (Workplace Dynamics)、决策风格 (Decision Making)、沟通风格 (Communication Style)、冲突解决 (Conflict Resolution)、项目执行 (Project Execution)、压力状态 (Work Under Pressure) 等共计 24 个常见职场与生活维度。
- **计分逻辑**：每道题有 4 个选项，分别对应 **D、I、S、C** 四种人格特征。系统记录各维度被选择的次数，并计算其占总题数的百分比：
  $$\text{维度得分 \%} = \text{round}\left( \frac{\text{该维度选择次数}}{24} \times 100 \right)$$
- **主导类型判定**：占比最高的维度即为用户的“主导性格类型”。若多个维度得分并列最高，按 D -> I -> S -> C 的优先级判定。

#### 3.2.2 页面组成与交互
- **顶部导航栏**：展示返回上一题的按钮，并实时呈现答题进度（如“第 3 / 24 题”）。
- **进度条**：在顶部导航栏下方显示全局线性进度条，答题时伴有平滑的伸缩动效。
- **题目主干区**：展示当前的评估类别（如 `DECISION MAKING`）与情境问题内容。
- **选项区域**：以单选卡片形式呈现 4 个选项，点击后高亮显示，并自动标记状态。
- **底部动作条**：
  - “帮助/提示”动作项。
  - “下一题”/“提交结果”按钮：只有用户已选择当前题目选项时才可点击（未选择时置灰）。点击后切到下一题，最后一题则为“提交结果”。

---

### 3.3 测评结果与分析模块 (Result Screen)
#### 3.3.1 结果总览 (Summary)
- 展示用户主导性格字母及中文名称（例如：**D (支配型)**）。
- 展示一句话的核心心理学画像总结。

#### 3.3.2 雷达图数据可视化 (Radar Chart)
- 采用轻量化、高性能的 SVG 绘制雷达极坐标图。
- 四个轴向分别映射 D, I, S, C 维度的占比（0% - 100%）。
- 绘制半透明蓝色遮罩区域，雷达图加载时伴随路径描边（Stroke-dasharray）缩放动效。
- 四角清晰展示 D, I, S, C 图例及个性化色彩。

#### 3.3.3 百分比得分条 (Dimension Progress)
- 以四色渐变条（红、黄、绿、蓝）直观展示用户四个维度的准确百分比。
- 页面加载时进度条实现从 0% 增长到目标值的动效。

#### 3.3.4 性格画像深度拆解 (Psychological Insights)
- **核心描述**：细化分析用户该性格的成因与职场特质。
- **双栏分析板**：
  - **核心优势 (Strengths)**：列出 3 条核心性格闪光点。
  - **提升领域 (Growth Opportunities)**：列出 3 条需要注意或提升的技能（如耐心、同理心、细节把握）。
- **底图插画**：在底部展示风格相符的团队管理/数据报表插画。

#### 3.3.5 行动唤起 (CTA Actions)
- 提供“查看深度解析”按钮，点击进入深度解析页。
- 提供“重新测试”和“查看测试历史”的双辅按钮。

---

### 3.4 深度解析模块 (Detailed Analysis)
#### 3.4.1 主要特质概览
- 在页面顶部展示主导类型的代表性图标（如 D 对应 `bolt` 闪电，I 对应 `groups` 社交，S 对应 `balance` 天平，C 对应 `fact_check` 对勾）与该维度的得分强度百分比进度条。

#### 3.4.2 结构化性格报告
- **核心优势卡片**：结合图标列出高阶实力。
- **职场表现卡片**：分析日常在公司的协作偏好、管理风格及对微观管理的态度。
- **沟通风格双向面板**：
  - **你的表达方式 (Express)**：如何向下、向同级传递信息。
  - **如何与你沟通 (Receive)**：其他人与你对话时的黄金法则与注意事项。
- **成长机会矩阵**：以三卡片矩阵形式列出针对主导性格瓶颈的微习惯建议。

#### 3.4.3 商业化/导出唤起 (Report Download) — [v2.0 修改]
- 原“下载完整报告”按钮由直接模拟下载修改为“付费后下载”流程（详见 3.7 支付系统）。

---

### 3.5 历史记录管理 (History Screen)
#### 3.5.1 数据持久化
- 每次完成测评后，系统将自动把带有 `id`, `date` (YYYY-MM-DD), `dominantType`, `scores`, `note` 的数据对象追加保存至浏览器的 `localStorage` 中（键名为 `disc_history`），若用户已登录，则异步同步至云端数据库。
- 首次加载时若无数据，应默认写入 3 条经典的模拟数据进行占位与展示。

#### 3.5.2 搜索与检索过滤
- 页面提供快捷搜索栏，支持根据以下关键词实时过滤：
  - 性格类型字母（如 `D`, `I`）。
  - 性格中文名（如 `支配型`, `稳健`）。
  - 日期（如 `2026年`, `3月`）。
  - 测评摘要备注内容。
- 支持一键清除搜索内容的 `x` 按钮。
- 无匹配记录时提供空状态提示及插图。

#### 3.5.3 记录操作
- **查看详情**：点击历史卡片内的“查看详情”按钮，即可将当前记录设为激活数据并载入结果页面。
- **记录删除**：提供删除按钮，点击后弹出二次确认弹窗。确认后从本地 `localStorage` 中移出该记录并实时刷新列表。

### 3.6 用户登录与注册模块 (Login & Register) — [v2.0 新增]
为了记录用户跨设备的历史数据及管理报告购买状态，系统需引入用户账户体系。

#### 3.6.1 本地登录与注册 (Local Authentication)
- **用户注册**：
  - 注册项：手机号/邮箱、密码、确认密码、验证码（图形/短信）。
  - 输入校验：手机号/邮箱格式校验，密码强度校验（至少6位，包含数字和字母）。
- **本地登录**：
  - 支持手机号/邮箱 + 密码登录。
  - 支持“记住我”功能（通过安全的 Cookie 或本地 Token 维持会话）。
  - 提供“忘记密码”重置通路。

#### 3.6.2 微信登录 (WeChat Authentication)
- **扫码登录（PC端）**：
  - 页面生成微信官方授权登录二维码。
  - 用户打开微信 App 扫码并确认授权。
  - 前端通过 WebSocket 或轮询机制监听扫码状态。授权成功后跳转并写入 Token。
- **微信内置浏览器登录（H5/移动端）**：
  - 自动静默获取微信微信授权或跳转至微信 OAuth 2.0 授权页面。
  - 微信授权回调后，后端自动解析 `code` 并获取微信用户的 `openid` 和 `unionid`。
  - 如果是首次登录，自动在系统后台静默创建账号或引导绑定已有手机号。

### 3.7 支付系统与报告下载流程 (Payment & Download) — [v2.0 新增]
深度报告采用付费解锁机制。

#### 3.7.1 支付限制与前置条件
- 当用户在“深度解析”页面点击“下载完整报告”时，系统首先判定当前测评记录的付费解锁状态（已付费或未付费）。
- 若用户未登录，强制引导其先完成登录/注册（本地或微信均可），以防付费记录丢失。
- 确认登录后，若未付费，则拉起支付收银台模态框。

#### 3.7.2 支付收银台 (Payment Modal)
- **支付方式支持**：微信支付、支付宝。
- **界面展示**：
  - 商品名称：DISC 职业性格 30 页深度分析报告（电子版）。
  - 商品价格：展示原价、限时特惠价及支付倒计时（如5分钟）。
  - 二维码区域：实时请求后端接口生成的动态支付二维码。
  - 状态文案：展示“请使用微信/支付宝扫码支付，支付成功后自动解锁下载”。
- **状态同步机制**：
  - 前端建立 WebSocket 链接或进行高频轮询（每3秒一次）检查该笔订单的支付状态。
  - 支付成功后，收银台模态框平滑淡出，界面直接变为“下载 PDF 报告”按钮。

#### 3.7.3 报告解锁与下载
- 后端在收到微信/支付宝支付网关的回调通知（Webhooks）后，将该用户该次测评记录的 `is_paid` 状态置为 `true`。
- 解锁后，用户可点击下载 PDF 报告。报告应由前端/后端渲染生成为美观的排版版面，包含行为盲点、管理建议与团队协作动态。

## 4. 非功能性与技术指标

### 4.1 技术栈要求
- **前端核心**：React 19+, Vite
- **样式系统**：Tailwind CSS v4 (Vanilla CSS 作为拓展)，应用遵循现代 HSL 渐变与 Material Design 3 风格。
- **字体系统**：英文字体采用 `Manrope`，代码及标签采用 `JetBrains Mono`。
- **图标系统**：使用 Google `Material Symbols Outlined` 矢量字体图标。

### 4.2 性能与兼容性
- **轻量无依赖**：核心功能（如雷达图）必须使用原生 SVG，不得引入庞大的第三方图表库，以保证极速加载和完全的样式定制。
- **响应式布局**：针对 Mobile、Tablet 和 PC 视口做极佳适配，底部导航栏在 PC 端隐藏并在 Mobile 端以底部 Tab Bar 呈现（仅在答题时隐藏，避免用户误触跳出）。
- **三方 SDK 接入**：合理引入微信开放平台 SDK，确保扫码及跳转授权的兼容性与流畅度。
- **安全与稳定性**：
  - 支付环节必须由后端签名交互，禁止前端明文存储任何密钥或订单敏感信息。
  - 登录 Token 需使用 JWT 进行签名加密，且支持过期与刷新机制。
